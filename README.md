# be-was-2024
코드스쿼드 백엔드 교육용 WAS 2024 개정판

## JON - 커스텀 자바 웹 서버

 순수 자바로 작성된 멀티 쓰레드 기반 HTTP 웹 서버 프로젝트 입니다

 이 프로젝트는 웹 애플리케이션 서버의 추상화 된 기능과 구조를 이해하기 위한 학습용 프로젝트임을 밝힙니다
 
### 핵심 기능

- 리플렉션 기반 컴포넌트 스캔
  - 스프링의 @Controller 를 모방한 간단한 컴포넌트 스캔을 실시합니다
  - @RequestMapping(method , path) 어노테이션을 서버 시작 전 webserver/handlers 패키지에서 읽어들입니다
  - 위 method + path 를 Map 에 키로, 그리고 대상 메서드를 값으로 매핑합니다
  - 완료된 Map 을 Router 클래스에 주입해 유저 요청을 처리합니다
- 싱글턴 구조의 Handlers 클래스
  - 스프링 컴포넌트들이 싱글턴으로 구성되는 것을 참조하여 핸들러를 가지는 클래스들은 리플렉션 동안 한번만 생성되어 반복 참조됩니다
  - 상태가 없도록 구성하여 멀티 쓰레드 환경에서도 싱글턴 참조에 문제가 없도록 설정하였습니다
- 가상 쓰레드 활용
  - newVirtualThreadPerTaskExecutor 메서드를 활용하여 요청과 가상 쓰레드의 1대1 매핑은 유지하되 OS 쓰레드의 Blocking을 최소화 했습니다
- 커스텀 HTTP 요청, 응답 클래스
  - HttpRequest, HttpResponse 클래스를 작성하고, 입출력 스트림을 통해 요청과 응답에 대응되는 객체를 사용합니다

### 서버 작동 흐름

1. 서버 부팅
   - ComponentScanner 가 지정 디렉토리를 읽고 클래스 로더와 리플렉션을 활용해 매핑으로 어노테이션 된 메서드를 맵에 저장해 반환합니다
   - Router 객체를 생성해 위 맵을 주입합니다
2. 유저 요청 대기 및 요청 처리
   - 서버에 요청이 도달하면 새로운 가상 쓰레드가 새로운 RequestHandler 를 생성해 Router 를 주입합니다
   - 요청이 파싱되어 라우터를 통해 각 url 을 담당하는 핸들러 클래스에 전달됩니다
   - 핸들러 클래스에서 최종 응답이 작성되어 유저에게 반환됩니다
